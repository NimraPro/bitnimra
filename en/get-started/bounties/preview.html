Development Bounties

Veil Developer Bounty Discord →

Development tasks that available for bounty rewards are defined as "bounty" tagged GitHub Issues in the Veil GitHub repository, and are listed below along with the price in Veil that has been assigned by the project.

The participation procedure is as follows:

  • Availability — Only tasks that are in the state available are available to be picked up for development. Tasks in other states have already been accepted and are being worked on.
  • Application — If you intend to work on an available task, post a note on the GitHub Issue, tagging the developer @presstab, and include an initial description of your approach to addressing the task.
  • Assignment — If your proposed approach is accepted, @presstab will make a note of that on the GitHub Issue, and will change the state to assigned.
  • Interaction — As you work on the task, you can post questions and feedback as notes on the issue. In addition, interactive discussion with the Veil development team is available through the dedicated Veil Development Discord
  • Progress — As the state of work progresses, you should tag @presstab in a note, requesting to update the state of work. The available states are: assigned, analysis, development, integration, testing and completed.
  • Completion & Payment — Once a task has been successfully completed, you will be paid the bounty in Veil.
